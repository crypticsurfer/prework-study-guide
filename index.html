HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Section elements are semantic elements that define a section in a document that contains thematic content.
  • An element refers to the entire element from opening tag to closing tag, whereas a tag refers only to what appears inside the angle brackets.
  • The p element represents a paragraph or block of text. You'll use this element extensively to wrap most of the text on your webpages.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.

CSS

  • CSS defines the colors, aesthetics, visual layout, and organization. We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • There are internal style sheets and external style sheets
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • complete git flow. git add -A, git commit -m "your message here", git pull origin main, git push origin "feature branch"

JavaScript

  • JavaScript is a powerful programming language that can add interactivity to a website.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • An operator is a mathematical symbol that produces a result based on two values (or variables).
  • Conditionals are code structures used to test if an expression returns true or not.
  • Functions are a way of packaging functionality that you wish to reuse.
